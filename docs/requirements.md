## Project Overview

Create a AI daily meal planner to help people maintain their health with ease.

## User Stories

### Authentication

- **US-01** — As a user, I want to register using email and password. ✅
- **US-02** — As a user, I want to log in with my credentials. ✅
- **US-03** — As a user, I want to log out. ✅

### Meal Plan Generation

- **US-04** — As a user, I want to enter my goal and health parameters.
- **US-05** — As a user, I want to submit the form and receive a daily meal plan generated by AI.

### Rate Limits

- **US-06** — As a user, I cannot generate more than 3 meal plans per day.
- **US-07** — As a user, I want to see my remaining requests for today.

### Security

- **US-08** — As a user, my password must be stored securely. ✅
- **US-09** — As a user, I cannot access the generation feature without logging in.

## Non-Functional Requirements

### Security (Technical)

- **NFR-01** — Passwords must be hashed. ✅
- **NFR-02** — All API endpoints except login/registration require session authentication. ✅

### Data

- **NFR-03** — Old meal plan requests are not stored.

## Out of Scope (for MVP)

- Payment system for premium plan
- Saving favorite meal plans
- Social sharing
- Google OAuth
